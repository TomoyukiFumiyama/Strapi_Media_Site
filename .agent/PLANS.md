# PLANS.md

## 目的
このファイルは**これから行う作業計画（マイルストーン）だけ**を管理する。完了済みタスクは本ファイルに残さず、履歴は `CHANGELOG.md` で管理する。

## 運用ルール
- 記載対象は未着手・進行中の計画のみ。
- タスク完了時は、該当項目を `PLANS.md` から削除する。
- 完了内容の記録は `CHANGELOG.md` に集約する。
- 常に「次に何をやるか」が一目で分かる状態を維持する。

## 驚きと発見（調査で分かったこと）
- 既存の `PLANS.md` は、途中でコードブロックの閉じ忘れと見出しレベル崩れがあり、Markdownレンダリングが壊れていた。
- 実装済み内容と今後計画が同居していたため、計画ファイルとしての即時性が低下していた。
- 本リポジトリでは履歴管理先として `CHANGELOG.md` が存在するため、`PLANS.md` は将来計画専用に分離した方が運用が安定する。

## マイルストーン（今後の作業計画）

### M1. コンテンツ運用の安定化
- Strapiの各主要コンテンツタイプで入力ガイド（必須/推奨）を明文化する。
- 公開前チェックリスト（SEO・relation・画像・導線）を運用手順に組み込む。

### M2. データ取得・整形の品質向上
- `lib/strapi` の query / mapper をページ用途（一覧/詳細）ごとに再点検し、取得項目を最適化する。
- null / relation未設定時のフォールバックを明示し、UI崩れを防ぐ。

### M3. SEO・再検証フローの強化
- metadata生成の優先順位（title / description / canonical / og）をテストで固定化する。
- Strapi更新時の再検証対象パスを整理し、過不足のない再生成運用を確立する。

### M4. テストカバレッジ拡張
- 主要mapperのユニットテストを追加する。
- local-pageのslug組み立てと`notFound()`分岐のテストを追加する。
- 主要ルートのmetadata生成テストを追加する。

### M5. ローカルSEO/資料導線の改善
- local-pageで地域固有情報の表示密度を高め、単純な地名差し替え化を防ぐ。
- blog / case-study / lead-magnet 間の導線relationを見直し、回遊性を改善する。
